## S4.02 Day-of Student Reg Opening {#s4.02-day-of-student-reg-opening}

**Owner:** Directors · **Last reviewed:** 2025 board (imported from the Hitchhiker's Guide, July 2026) · **Status:** current — when you find something stale, fix it and update this line.

When student reg opens, you should expect to spend about 4 hours answering emails. Inevitably the website becomes extremely slow, and you will not be able to access any features. If you tested student reg, there shouldn’t be any major technical glitches. Despite this numerous parents will send emails claiming that the website is broken. You should be coordinating with your co-director about who will be answering which emails, and addressing issues that arise with your mentor, if they are not easily solved. A few policies to keep in mind:

* Even if parents/kids complain that they waited an hour for the website to load and now their favorite class has no spots left, you should not put them into the class  
* Parents/kids will oftentimes say that the website won’t load and ask us to register for them. Do not do this--reply and say that the student has to register.  
* As appropriate, emphasize that students (not parents) should be the ones signing up for classes

Take a look at some of the example case emails below:

| Scenario 1: Parent and/or student had the website crash |
| :---- |
| Times you might receive this: During student reg |
| Policy: Due to high traffic, this happens a lot in student reg. |
| Email: *“Hello, I am trying to register for Splash, but the site crashed after I clicked the lunch options.  Should I keep reloading the page?  Thank you so much for your help\! I’m really excited about attending Splash.”*  |
| Sample Response to student: Hello, Due to heavy traffic on the website, the links may temporarily be down. Sorry for this inconvenience--please try the links again in 5-10 minutes and hopefully they will work for you\! Thank you for your patience. Spring Splash Co-Director  |

| Scenario 2: Parent and/or student is experiencing difficulty logging in during student registration |
| :---- |
| Times you might receive this: During student reg |
| Policy: Weird website things happen pretty often. They usually resolve themselves. |
| Email: *“My sister just registered and had no trouble but I have been unable to register, every time I try to create a sign up it gives me an oops message that says to make sure cookies are enabled (they are).  This happened on both my laptop and my parents' desktop.  It seems to have created a login for me but didn’t take any other info and won’t let me log in.  I want to make sure I get classes I want but don’t know how. “* |
| Sample Response to student:   Due to heavy traffic on the website, the website may temporarily be down. Sorry for this inconvenience--please try the site again in five minutes or so and hopefully it will work. Please let us know if you have any other questions\! Best, Splash Team |
| *Follow-up student email: “OK, thanks.  I just got this though. Forbidden (403) CSRF verification failed. Request aborted. You need to ensure: Your browser is accepting cookies. Your browser has Javascript enabled. If you are certain you have done all of the above, this probably means we messed up.”* |
| Sample response: We recommend either enabling cookies on this browser or trying to register on a different browser. Please let us know if you have any other questions\! Best, Splash Team |

| Scenario 3: Parent and/or student wants to be put on a waiting list |
| :---- |
| Times you might receive this: During student reg |
| Policy: We don’t have waiting lists, but students can try to switch into the class on the day of. |
| Email: *“Hi, I really want to do Crazy Experiments in Genetics but it was full. If possible please add me to the list for the 1:30 pm \- 2.25 pm slot.”* |
| Sample Response to student:   Hello, While we don't have waitlists for classes, spots normally open up on the day of the program, so don't worry\! On the day of the program you will have an opportunity to make that change if a seat opens up. Thanks, Spring Splash Co-director  |

| Scenario 4: Parent and/or student wants to view their schedule and/or confirm their registration.  |
| :---- |
| Times you might receive this: During/after student reg |
| Policy: You shouldn’t send them their schedule. Just send them the link so they can access it. This is for two reasons (1) we want to be sure it’s the right person and (2) the schedules that we have for the students will have room numbers, which you shouldn’t give out before the program (it de-incentivizes signing in). |
| Email: *“*Hi,  My children told me they registered for Splash yesterday, but we did not receive any email confirmation nor was I able to see their schedule online.Their names are Andrei Zubek and Petru Zubek. Can you please confirm their registration and send me their schedule? Thank you, Valentina” |
| Sample Response to student:   Hello, Both of your children have registered. In order to view their schedules, please have your children go to the link below and click on the blue "view receipt" button at the bottom of the page. Let me know if you have any questions\! [https://yale.learningu.org/learn/Splash/2019\_Spring/studentreg](https://yale.learningu.org/learn/Splash/2019_Spring/studentreg) Thank you, Spring Splash Co-Director  |
