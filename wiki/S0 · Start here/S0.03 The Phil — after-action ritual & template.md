## S0.03 The Phil — after-action ritual & template

**Owner:** Directors (program Phils) & Presidents (year-end SuperPHIL) · **Last reviewed:** July 2026 · **Status:** current — when you find something stale, fix it and update this line.

### What it is

The Phil is Splash's after-action review. One happens after **every program** — Splash and Sprout, each semester, no exceptions — while memories are fresh (within two weeks of the program day). The presidents run a year-end **SuperPHIL** covering the whole academic year. Phils are the intake valve of the org's permanent record: the wiki only stays alive because Phils feed it.

### How to run one

1. Book 60–90 minutes with the full admin team. Food helps attendance.
2. Directors pre-fill the "Numbers" section and skim last cycle's Phil beforehand — check whether its "do differently" items actually happened.
3. Walk the template top to bottom. Capture disagreements as written alternatives rather than debating to consensus.
4. **Before ending the meeting**, fill in the last section (wiki edits) and assign each edit to a named person with a one-week deadline. This is the step that makes the other steps matter.
5. Save the doc as `Phil — {program}.gdoc` (e.g., `Phil — F26 Splash`) in the program's folder in `1 · Current Semester`. It gets archived with the program folder.

### After the Phil

The directors make the assigned wiki edits (`wiki/` in the Handbook folder → edit the page → update its "Last reviewed" line → run `python3 rebuild.py` → re-upload `Splash HQ.html`). If you can't run the rebuild, make the edits anyway and leave the rebuild for whoever can — the wiki files are the source of truth, the app is just the reader.
